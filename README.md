# Green Agro Seeds — Sales Management System

একটা premium, responsive সেলস ম্যানেজমেন্ট ওয়েব অ্যাপ — এখন সম্পূর্ণ **offline (localStorage)** কাজ করে, পরে চাইলে Google Sheets-এ cloud sync যোগ করা যাবে।

## ফাইল স্ট্রাকচার

```
green-agro-seeds/
├── index.html              ← মূল অ্যাপ শেল, সব পেজ + মোডাল এখানে
├── css/
│   ├── style.css           ← ডিজাইন সিস্টেম (রঙ লোগো থেকে: #136831 সবুজ, #FDFEF1 ক্রিম)
│   └── pages.css           ← পেজ-ভিত্তিক লেআউট
├── data/
│   └── products.js         ← পণ্য ক্যাটালগ (মূল্য তালিকা ২০২৬ থেকে হুবহু তোলা)
├── js/
│   ├── pricing.js          ← প্যাকেট-বেসিস / বাল্ক-বেসিস দাম হিসাবের ইঞ্জিন
│   ├── customers.js        ← গ্রাহকের বাকি/পাওনার running-balance লেজার
│   ├── history.js          ← বিক্রয় ইতিহাস + পণ্য তালিকা রেন্ডারিং
│   ├── pdf.js               ← Invoice (দামসহ) ও Chalan (দাম ছাড়া) PDF জেনারেশন
│   └── app.js               ← নেভিগেশন, বিক্রয় ফর্ম, মূল লজিক
└── google-apps-script.js   ← (ঐচ্ছিক, পরে ব্যবহারের জন্য) Google Sheets ব্যাকএন্ড কোড
```

## যেভাবে চালাবেন

শুধু `index.html` ব্রাউজারে খুললেই চলবে। ইন্টারনেট শুধু Google Fonts আর PDF লাইব্রেরি (CDN) লোড করতে লাগবে; ডেটা সবসময় স্থানীয়ভাবে (browser-এর localStorage) সেভ হয়।

Netlify-তে ডিপ্লয় করতে চাইলে পুরো ফোল্ডারটা drag-and-drop করলেই হবে (Khalid's Dreams-এর মতোই)।

## মূল্য হিসাবের লজিক

প্রতিটা পণ্যের জাত ও পরিমাণ (quantity option)-এর জন্য ৪টা দাম শীট থেকে হুবহু সংরক্ষিত আছে — প্যাকেট, কেজি, বাল্ক/লুজ, খুচরা। এগুলো ফর্মুলা দিয়ে অটো-ক্যালকুলেট করা হয়নি, কারণ কিছু পণ্যে (যেমন তরমুজ) দাম রৈখিক সূত্র মানে না। বিক্রয়ের সময়:

- **প্যাকেট মোডে:** মোট = প্যাকেট দাম × প্যাকেট সংখ্যা
- **বাল্ক মোডে:** মোট = (কেজি দাম বা বাল্ক দাম) × মোট কেজি

নতুন পণ্য/জাত "পণ্য তালিকা" পেজ থেকে ম্যানুয়ালি যোগ করা যায় — কোড টাচ না করেই।

## বাকি/পাওনার হিসাব (Ledger)

সিম্পল single-entry running balance:
- বিক্রয়ে বাকি থাকলে (paid < total) স্বয়ংক্রিয়ভাবে সেই গ্রাহকের নামে **ডেবিট** এন্ট্রি যোগ হয়
- "বাকি/পাওনা" পেজ থেকে ম্যানুয়ালি ডেবিট (নতুন বাকি) বা ক্রেডিট (পরিশোধ) এন্ট্রি যোগ করা যায়
- ব্যালান্স = মোট ডেবিট − মোট ক্রেডিট (ধনাত্মক মানে গ্রাহক বাকি আছেন)

## দুই ধরনের PDF

1. **ইনভয়েস** — সম্পূর্ণ দামসহ বিল
2. **চালান** — শুধু পণ্য, পরিমাণ, মোট ওজন ও গ্রাহকের নাম/ঠিকানা — কোনো দাম নেই

## পরবর্তী ধাপ (যখন প্রয়োজন হবে)

- **Google Sheets cloud sync:** `google-apps-script.js` ফাইলটা একটা Google Sheet-এ Apps Script হিসেবে ডিপ্লয় করে `js/app.js`-এ fetch কল যোগ করলেই একাধিক ডিভাইস থেকে ডেটা সিঙ্ক হবে।
- **সাপ্লায়ার/ডিলার পক্ষের হিসাব:** ঠিক একই ledger প্যাটার্ন সম্প্রসারণ করে যোগ করা যাবে।
- **স্থায়ী ব্র্যান্ড লোগো ফাইল:** এখন CSS-এ আঁকা leaf আইকন ব্যবহার হয়েছে; চাইলে আসল লোগো ইমেজ (`logo.jpeg`) বসিয়ে দেওয়া যাবে।
